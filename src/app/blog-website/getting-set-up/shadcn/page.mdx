## Initializing `shadcn/ui`

To install the `shadcn/ui` library, which provides a set of accessible and customizable UI components for React, run the following command in your terminal:

```bash
npx shadcn@latest init
```

This command will set up the `shadcn/ui` library in your Next.js project, allowing you to use its components in your portfolio website.

It will prompt you to choose a base color for your Tailwind CSS configuration. You can select from the following options:


### `src/lib/utils.ts` File

It will also create your `lib/` directory, which contains utility functions and hooks for your project. For right now, it will contain a `utils.ts` file with some utility functions that you can use in your project.

The file will look like this:

```typescript
// src/lib/utils.ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

This utility function `cn` is used to merge class names together, which is useful for styling components with Tailwind CSS.

For example, you can use it to conditionally apply classes to your components based on certain conditions.

```typescript
// src/components/ui/button.tsx
import { cn } from "@/lib/utils";
const Button = ({ primary, className }) => {
  return (
    <button className={cn("btn", primary && "btn-primary", className)}>
      Click Me
    </button>
  );
};
```

### About oklch Colors

The `shadcn/ui` library uses the `oklch` color space for its color variables. The `oklch` color space is a perceptually uniform color space that provides a more consistent and intuitive way to work with colors compared to traditional RGB or HSL color spaces.

If you're more comfortable with hex or RGB colors, you can easily convert `oklch` colors to those formats using online tools or libraries. However, it's recommended to use `oklch` colors when working with the `shadcn/ui` library to take advantage of its design system and ensure consistency across your project.
The `oklch` color space is based on the CIELAB color space, which is designed to be more perceptually uniform. This means that colors in the `oklch` space are more consistent in terms of how they appear to the human eye, regardless of the device or lighting conditions.

The `oklch` color space is defined by three components: lightness (L), chroma (C), and hue (H). The lightness component represents how light or dark a color is, the chroma component represents the intensity or saturation of the color, and the hue component represents the color itself (e.g., red, green, blue).

The `oklch` color space is designed to be more intuitive and easier to work with than traditional RGB or HSL color spaces. It allows designers and developers to create colors that are more visually consistent and accessible, making it easier to create designs that look good across different devices and lighting conditions

You can learn more about the `oklch` color space and how to use it in your designs by visiting the [oklch documentation](https://www.w3.org/TR/css-color-4/#oklch).

Maybe you're wondering why the `shadcn/ui` library uses `oklch` colors instead of the more traditional RGB or HSL colors. The reason is that `oklch` colors are designed to be more perceptually uniform, meaning that they look more consistent across different devices and lighting conditions.

## Downloading the Shadcn/ui Components

To download the `shadcn/ui` components, you can use the `npx shadcn@latest add` command followed by the component name. For example, to add the `Button` component, you would run:

```bash
npx shadcn@latest add button
```

This command will download the `Button` component and its dependencies, and add it to your project.

You can also add multiple components at once by separating their names with a space. For example, to add the `Button`, `Card`, and `Input` components, you would run:

```bash
npx shadcn@latest add button card input
```

For this portfolio website, we will be adding the following components:

- `Accordion`
- `Badge`
- `Breadcrumb`
- `Button`
- `Card`
- `Carousel`
- `Checkbox`

You can add all of these components at once by running the following command:

```bash
npx shadcn@latest add accordion badge breadcrumb button card carousel checkbox dialog dropdown-menu input label navigation-menu select separator sidebar sonner switch tabs textarea toggle tooltip
```

### Using the Components

Once you've added the components, you can start using them in your project. Each component will have its own directory with a `ComponentName.tsx` file and a `ComponentName.css` file.

You can then use these components in your JSX like this:

```jsx
// src/components/ui/button.tsx
import { Button } from "@/components/ui/button";

export function ButtonSecondary() {
  return <Button>Button</Button>;
}
```

```jsx
// src/components/ui/card.tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardAction,
  CardContent,
  CardFooter,
} from "@/components/ui/card";

export function CardDemo() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
        <CardDescription>Card Description</CardDescription>
        <CardAction>Card Action</CardAction>
      </CardHeader>
      <CardContent>
        <p>Card Content</p>
      </CardContent>
      <CardFooter>
        <p>Card Footer</p>
      </CardFooter>
    </Card>
  );
}
```

```jsx
// src/components/ui/input.tsx
import { Input } from "@/components/ui/input";
export function InputDemo() {
  return <Input type="email" placeholder="Email" />;
}
```

```jsx
//  src/components/ui/input-file.tsx
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";

export function InputFile() {
  return (
    <div className="grid w-full max-w-sm items-center gap-3">
      <Label htmlFor="picture">Picture</Label>
      <Input id="picture" type="file" />
    </div>
  );
}
```

This will render the `Button`, `Card`, and `Input` components with the styles and functionality provided by the `shadcn/ui` library.

### Customizing the Components

You can customize the components by passing props to them. Each component will have its own set of props that you can use to modify its behavior and appearance.
For example, the `Button` component might have a `variant` prop that allows you to change its style:

```jsx
// src/components/ui/button.tsx
import { Button } from "@/components/ui/button";
export function ButtonSecondary() {
  return <Button variant="secondary">Secondary</Button>;
}
```

You can also use the `className` prop to apply additional styles to the components:

```jsx
// src/components/ui/button.tsx
import { Button } from "@/components/ui/button";
export function ButtonSecondary() {
  return <Button className="bg-blue-500 text-white">Secondary</Button>;
}
```

You can also include the components in your pages or other components by importing them and using them in your JSX. For example, you can create a `Button` component like this:

```jsx
// src/components/ui/button-with-icon.tsx
import { IconGitBranch } from "@tabler/icons-react";

import { Button } from "@/components/ui/button";

export function ButtonWithIcon() {
  return (
    <Button variant="outline" size="sm">
      <IconGitBranch /> New Branch
    </Button>
  );
}
```

This will render a button with an icon and some text. You can use any of the icons from the `@tabler/icons-react` library, or any other icon library you prefer.

You can also pass additional props to the components to customize their behavior. For example, you can pass an `onClick` prop to the `Button` component to handle click events:

```jsx
// src/components/ui/button-with-click.tsx
import { Button } from "@/components/ui/button";
export function ButtonWithClick() {
  const handleClick = () => {
    console.log("Button clicked!");
  };
  return <Button onClick={handleClick}>Click Me</Button>;
}
```
