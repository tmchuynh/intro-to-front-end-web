## Understanding MDX Blog Post Structure

MDX blog posts combine the simplicity of Markdown with the power of React components. Each blog post is a `.mdx` file that contains frontmatter (metadata) and content written in MDX syntax.

### File Organization

Your blog posts should be organized in a clear directory structure:

```txt title="File Structure"
content/
└── posts/
    ├── getting-started-with-react.mdx
    ├── advanced-typescript-tips.mdx
    ├── web-performance-optimization.mdx
    └── categories/
        ├── tutorials/
        │   ├── building-your-first-app.mdx
        │   └── deploying-to-vercel.mdx
        └── reviews/
            ├── framework-comparison.mdx
            └── tool-reviews.mdx
```

## Frontmatter: the Metadata Foundation

Frontmatter is the YAML section at the beginning of your MDX file that contains metadata about your blog post. This information is used for SEO, sorting, categorization, and display purposes.

### Basic Frontmatter Structure

```yaml
---
title: "Your Amazing Blog Post Title"
description: "A compelling description that summarizes your post content"
date: "2024-12-15"
author: "Your Name"
category: "Web Development"
tags: ["React", "TypeScript", "Next.js"]
featured: false
published: true
image: "/images/blog/your-post-image.jpg"
imageAlt: "Descriptive alt text for your image"
readingTime: 8
---
```

### Essential Frontmatter Fields

| Field         | Type    | Description                                      | Required |
| ------------- | ------- | ------------------------------------------------ | -------- |
| `title`       | string  | The main title of your blog post                 |          |
| `description` | string  | Meta description for SEO and previews            |          |
| `date`        | string  | Publication date (YYYY-MM-DD format)             |          |
| `author`      | string  | Author name                                      |          |
| `category`    | string  | Primary category for organization                |          |
| `tags`        | array   | Array of relevant tags for filtering             |          |
| `published`   | boolean | Whether the post is published (draft if false)   |          |
| `featured`    | boolean | Mark as featured post                            |          |
| `image`       | string  | Path to featured image                           |          |
| `imageAlt`    | string  | Alt text for featured image                      |          |
| `readingTime` | number  | Estimated reading time in minutes                |          |
| `updatedAt`   | string  | Last update date                                 |          |
| `slug`        | string  | Custom URL slug (auto-generated if not provided) |          |
| `canonical`   | string  | Canonical URL if republished content             |          |

#### 3. Code Examples with Context

Always provide context for your code examples:

Let's create a simple `Button` component that accepts different variants:

```tsx
// components/Button.tsx
import React from "react";
import { cn } from "@/lib/utils";

interface ButtonProps {
  variant?: "primary" | "secondary" | "outline";
  size?: "sm" | "md" | "lg";
  children: React.ReactNode;
  onClick?: () => void;
  disabled?: boolean;
}

export const Button: React.FC<ButtonProps> = ({
  variant = "primary",
  size = "md",
  children,
  onClick,
  disabled = false,
}) => {
  const baseClasses = "font-medium rounded-lg transition-colors";

  const variantClasses = {
    primary: "bg-blue-600 text-white hover:bg-blue-700",
    secondary: "bg-gray-600 text-white hover:bg-gray-700",
    outline: "border-2 border-blue-600 text-blue-600 hover:bg-blue-50",
  };

  const sizeClasses = {
    sm: "px-3 py-1.5 text-sm",
    md: "px-4 py-2 text-base",
    lg: "px-6 py-3 text-lg",
  };

  return (
    <button
      className={cn(
        baseClasses,
        variantClasses[variant],
        sizeClasses[size],
        disabled && "opacity-50 cursor-not-allowed"
      )}
      onClick={onClick}
      disabled={disabled}
    >
      {children}
    </button>
  );
};
```

This component demonstrates several React best practices:

- TypeScript interfaces for type safety
- Default parameter values for better developer experience
- Conditional styling using utility classes
- Props destructuring for cleaner code

## Writing Best Practices

### Content Guidelines

#### 1. Write for Your Audience

- Beginner-friendly: Explain concepts clearly and avoid jargon
- Provide context: Don't assume prior knowledge
- Use examples: Show, don't just tell

````mdx showLineNumbers=false
// Bad: Assumes knowledge
Use useCallback to memoize functions.

// Good: Provides context and examples
The `useCallback` hook helps optimize performance by memoizing functions, preventing unnecessary re-renders of child components.

```tsx
// Without useCallback - function recreated on every render
const handleClick = () => {
  console.log("Button clicked");
};

// With useCallback - function memoized
const handleClick = useCallback(() => {
  console.log("Button clicked");
}, []); // Empty dependency array means function never changes
```
````

#### 2. Be Concise but Comprehensive

- Get to the point quickly
- Provide enough detail to be useful
- Use bullet points and lists for clarity

#### 3. Include Real-World Examples

- Show how concepts apply in practice
- Use relatable scenarios
- Demonstrate with complete code snippets

### Technical Writing Tips

#### 1. Code Quality

- Syntax highlighting: Always specify the language for code blocks
- Complete examples: Provide working, copy-pasteable code
- Error handling: Show how to handle edge cases
- Comments: Explain complex logic inline

```tsx
//  Good: Complete, commented example
import { useState, useEffect } from "react";
import { toast } from "sonner";

interface User {
  id: string;
  name: string;
  email: string;
}

export const UserProfile = ({ userId }: { userId: string }) => {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchUser = async () => {
      try {
        setLoading(true);
        setError(null);

        const response = await fetch(`/api/users/${userId}`);

        if (!response.ok) {
          throw new Error("Failed to fetch user");
        }

        const userData = await response.json();
        setUser(userData);
      } catch (err) {
        const errorMessage =
          err instanceof Error ? err.message : "Unknown error";
        setError(errorMessage);
        toast.error(`Error loading user: ${errorMessage}`);
      } finally {
        setLoading(false);
      }
    };

    fetchUser();
  }, [userId]); // Re-fetch when userId changes

  // Loading state
  if (loading) {
    return <div className="animate-pulse">Loading user...</div>;
  }

  // Error state
  if (error) {
    return (
      <div className="text-red-600">
        <p>Error: {error}</p>
        <button onClick={() => window.location.reload()}>Try Again</button>
      </div>
    );
  }

  // Success state
  return (
    <div className="bg-white p-6 rounded-lg shadow">
      <h2 className="text-xl font-bold">{user?.name}</h2>
      <p className="text-gray-600">{user?.email}</p>
    </div>
  );
};
```
