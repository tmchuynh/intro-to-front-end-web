## Using the Hooks and Storing the Data

```tsx
// src/app/checkout/confirmation/page.tsx
function OrderConfirmationContent() {
  const router = useRouter();
  const searchParams = useSearchParams();
  const orderId = searchParams.get("orderId");
  const { getOrderById } = useOrder();
  const { formatPrice } = useCurrency();
  const { removeDiscount } = useCart();

  const [order, setOrder] = useState<Order | null>(null);
  const [loading, setLoading] = useState(true);
```

### Explanation of the Code

This code imports necessary hooks and components, sets up the `OrderConfirmationContent` component, and initializes state variables for managing the order data and loading state. The `useRouter` and `useSearchParams` hooks are used to access the current route and query parameters, specifically to retrieve the `orderId`.

## `useEffect` Hook

```tsx
// src/app/checkout/confirmation/page.tsx
useEffect(() => {
  if (orderId) {
    removeDiscount(); // Clear any existing discounts
    const foundOrder = getOrderById(orderId);
    if (foundOrder) {
      setOrder(foundOrder);
    } else {
      toast.error("Order not found");
      router.push("/");
    }
  } else {
    router.push("/");
  }
  setLoading(false);
}, [orderId, getOrderById, router]);
```

### Explanation of the Code

This `useEffect` hook runs when the component mounts or when the `orderId` changes. It checks if an `orderId` is present in the query parameters. If it is, it attempts to retrieve the order using the `getOrderById` function. If the order is found, it sets the order state; otherwise, it displays an error message and redirects the user to the home page. The loading state is set to false after attempting to fetch the order.

## The Tracking Number

const copyTrackingNumber = () => {
  if (order?.trackingNumber) {
    navigator.clipboard.writeText(order.trackingNumber);
    toast.success("Tracking number copied to clipboard!");
  }
};

### Explanation of the Code

This function allows users to copy the tracking number to their clipboard. If the order has a tracking number, it uses the `navigator.clipboard.writeText` method to copy it and displays a success message using `toast`.

## Implementing Downloadable Invoice

```tsx
// src/app/checkout/confirmation/page.tsx
const downloadReceipt = () => {
  if (!order) return;
  const receiptData = `
      Order ID: ${order.id}
      Order Date: ${new Date(order.orderDate).toLocaleDateString("en-US")}
      Tracking Number: ${order.trackingNumber}
      Customer: ${order.customerName} (${order.customerEmail})
      Shipping Address: ${order.shippingAddress}
      Payment Method: ${order.paymentMethod}
      Status: ${order.status}
      Subtotal: ${formatPrice(order.totalAmount)}
      Discount: ${formatPrice(order.discountAmount)}
      Membership Discount: ${formatPrice(order.membershipDiscount ?? 0)}
      Shipping Fee: ${
        order.shippingFee > 0 ? formatPrice(order.shippingFee) : "FREE"
      }
      Total Amount: ${formatPrice(order.grandTotal)}
      Items: ${order.items
        .map((item) => `${item.title} (x${item.quantity})`)
        .join(", ")}
    `;
  const blob = new Blob([receiptData], { type: "text/plain" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `receipt-${order.id}.txt`;
  a.click();
  URL.revokeObjectURL(url);
};
```

### Explanation of the Code

This function generates a downloadable receipt in plain text format. It compiles the order details into a string, creates a Blob from that string, and then creates a temporary link to download the file. When the user clicks the link, the receipt is downloaded with a filename based on the order ID.

## Loading Fallback Function

```tsx
// src/app/checkout/confirmation/page.tsx
function LoadingFallback() {
  return (
    <div className="flex justify-center items-center bg-background min-h-screen">
      <div className="text-center">
        <div className="mx-auto mb-4 border-primary border-b-2 rounded-full w-8 h-8 animate-spin"></div>
        <p className="text-muted-foreground">Loading order confirmation...</p>
      </div>
    </div>
  );
}
```

### Explanation of the Code

This function provides a loading fallback UI that displays a spinner and a loading message while the order details are being fetched. It ensures that users have a visual indication that their request is being processed.

## Rendering the Confirmation Page

```tsx
// src/app/checkout/confirmation/page.tsx
export default function OrderConfirmationPage() {
  return (
    <Suspense fallback={<LoadingFallback />}>
      <OrderConfirmationContent />
    </Suspense>
  );
}
```

### Explanation of the Code

This code exports the main `OrderConfirmationPage` component, which uses React's `Suspense` to handle the loading state. The `OrderConfirmationContent` component is wrapped in a `Suspense` component, allowing it to display the `LoadingFallback` while the order data is being fetched.
