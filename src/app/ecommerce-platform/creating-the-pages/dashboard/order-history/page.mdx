## Filtering and Sorting Orders

```tsx
// src/app/dashboard/my-orders/page.tsx
const filteredOrders = orders
  .filter((order) => {
    const matchesSearch =
      order.id.toLowerCase().includes(searchTerm.toLowerCase()) ||
      order.trackingNumber.toLowerCase().includes(searchTerm.toLowerCase()) ||
      order.items.some((item) =>
        item.title.toLowerCase().includes(searchTerm.toLowerCase())
      );
    const matchesStatus =
      statusFilter === "all" || order.status === statusFilter;
    return matchesSearch && matchesStatus;
  })
  .sort((a, b) => {
    switch (sortBy) {
      case "date-desc":
        return (
          new Date(b.orderDate).getTime() - new Date(a.orderDate).getTime()
        );
      case "date-asc":
        return (
          new Date(a.orderDate).getTime() - new Date(b.orderDate).getTime()
        );
      case "amount-desc":
        return b.grandTotal - a.grandTotal;
      case "amount-asc":
        return a.grandTotal - b.grandTotal;
      default:
        return 0;
    }
  });
```

### Explanation of the Code

The code above filters the orders based on the search term and status filter, and sorts them according to the selected criteria. It uses the `filter` method to check if each order matches the search term or status, and the `sort` method to arrange the orders based on date or amount.

## Handleling Order Expansion

```tsx
// src/app/dashboard/my-orders/page.tsx
const toggleOrderExpansion = (orderId: string) => {
  const newExpanded = new Set(expandedOrders);
  if (newExpanded.has(orderId)) {
    newExpanded.delete(orderId);
  } else {
    newExpanded.add(orderId);
  }
  setExpandedOrders(newExpanded);
};
```

### Explanation of the Code

The `toggleOrderExpansion` function manages the expansion state of each order. It uses a `Set` to track which orders are expanded, allowing for efficient addition and removal of order IDs when the user clicks to view more details.
