## Fetching All Products and Organizing by Categories

```tsx startLineNumber=37
// src/app/shopping/categories/page.tsx
useEffect(() => {
  const loadProducts = async () => {
    try {
      setLoading(true);
      const allProducts = await getAllProducts();
      setProducts(allProducts);

      // Group products by category
      const categoryMap = new Map<string, ProductItem[]>();
      allProducts.forEach((product) => {
        if (!categoryMap.has(product.category)) {
          categoryMap.set(product.category, []);
        }
        categoryMap.get(product.category)!.push(product);
      });

      // Create category data with counts
      const categoryData: CategoryData[] = Array.from(categoryMap.entries())
        .map(([category, categoryProducts]) => ({
          id: category,
          name: category
            .replace(/-/g, " ")
            .replace(/\b\w/g, (l) => l.toUpperCase()),
          count: categoryProducts.length,
          products: categoryProducts,
        }))
        .sort((a, b) => a.name.localeCompare(b.name));

      setCategories(categoryData);
    } catch (err) {
      setError(
        err instanceof Error ? err.message : "Failed to load categories"
      );
    } finally {
      setLoading(false);
    }
  };

  loadProducts();
}, []);

const selectedCategoryData = categories.find(
  (cat) => cat.id === selectedCategory
);
```

##### Explanation of the Code

This code fetches all products from the API and organizes them into categories. It uses a `Map` to group products by their category, creating a structured `CategoryData` array that includes the category name, product count, and associated products. The categories are sorted alphabetically for better user experience.

##### Explanation of the Code

This section renders the product cards based on the selected category or displays an overview of all categories. It includes a hero section, category statistics, and a grid of categories. When a category is selected, it shows the products within that category in either grid or list view, depending on the user's choice.

The category stats include total categories, total products, and the largest category by item count. Each category card displays a sample product image, category name, product count, and popular items within that category.

The category grid is responsive, adapting to different screen sizes, and includes hover effects for better user interaction. The quick actions section at the bottom provides links to browse all products, view sale items, and check new arrivals.

the category products view includes a back button to return to the category overview, a toggle for view modes (grid or list), and displays the products in the selected category.

The products display is handled by the `ProductGrid` component, which takes care of rendering the products based on the selected view mode.

The quick actions section at the bottom provides buttons for users to browse all products, view sale items, and check new arrivals, enhancing user navigation and engagement.
