# The Contact Page

## Table of Contents

## Overview

### Purpose

The Contact page is designed to provide users with a way to get in touch with the business. It typically includes a contact form, phone number, email address, and physical address.

### Content

The Contact page should include the following elements:

- A contact form for users to fill out
- A phone number for direct contact
- An email address for inquiries
- A physical address for the business
- A map showing the business location (optional)

### Design

The design of the Contact page should be clean and easy to navigate. It should include:

- A clear heading indicating the purpose of the page
- A simple and intuitive contact form
- Contact information displayed prominently
- A map (if included) that is easy to read and understand

### Accessibility

Ensure that the Contact page is accessible to all users, including those with disabilities. This includes:

- Using semantic HTML elements for the contact form
- Providing alternative text for images (e.g., the map)
- Ensuring that the page is navigable using a keyboard
- Using ARIA roles and attributes where necessary

### Mobile Responsiveness

Ensure that the Contact page is mobile-responsive, meaning it should:

- Display correctly on various screen sizes
- Allow users to easily fill out the contact form on mobile devices
- Ensure that contact information is easily readable on smaller screens

The `ContactFormData` interface is used to type the form data, ensuring that all required fields are present.

## Real-Time Validation

```tsx
// src/lib/utils/contact.ts
const validateField = useCallback(
  (field: keyof ContactFormData, value: string) => {
    const error = validateContactField(field, value, formData);
    setErrors((prev) => ({
      ...prev,
      [field]: error || "",
    }));
    return !error;
  },
  [formData]
);
```

### Explanation of the Code

The `validateField` function is used to validate individual fields in the contact form in real-time. It takes the field name and its value as parameters, validates the field using the `validateContactField` function, and updates the errors state accordingly. This allows for immediate feedback to the user as they fill out the form.

## Handling Input Changes

```tsx
// src/app/contact-us/page.tsx
const handleInputChange = (
  e: React.ChangeEvent<HTMLInputElement | HTMLTextAreaElement>
) => {
  const { name, value } = e.target;
  const fieldName = name as keyof ContactFormData;

  setFormData((prev) => ({
    ...prev,
    [fieldName]: value,
  }));

  // Validate field on blur or after user stops typing
  setTimeout(() => validateField(fieldName, value), 300);
};

const handleSelectChange = (value: string) => {
  setFormData((prev) => ({
    ...prev,
    category: value,
  }));
  validateField("category", value);
};

const handleBlur = (
  e: React.FocusEvent<HTMLInputElement | HTMLTextAreaElement>
) => {
  const { name, value } = e.target;
  validateField(name as keyof ContactFormData, value);
};
```

### Explanation of the Code

The `handleInputChange` function updates the form data state when the user types in the input fields. It also validates the field after a short delay (300 milliseconds) to avoid excessive validation calls while the user is typing. The `handleSelectChange` function updates the category field when the user selects an option from the dropdown, and the `handleBlur` function validates the field when it loses focus.

## Submitting the Form

```tsx
// src/app/contact-us/page.tsx
const handleSubmit = async (e: React.FormEvent) => {
  e.preventDefault();
  setIsSubmitting(true);
  setSubmitStatus("idle");

  // Comprehensive form validation
  const validation = validateContactForm(formData);

  if (!validation.isValid) {
    // Set all validation errors
    const newErrors: Record<keyof ContactFormData, string> = {
      name: "",
      email: "",
      subject: "",
      category: "",
      message: "",
    };

    validation.errors.forEach((error: ContactValidationError) => {
      newErrors[error.field] = error.message;
    });

    setErrors(newErrors);
    setIsSubmitting(false);
    setSubmitStatus("error");
    return;
  }

  try {
    // Sanitize form data before submission
    const sanitizedData = sanitizeContactForm(formData);

    // Simulate API call
    await new Promise((resolve) => setTimeout(resolve, 2000));

    console.log("Form submitted:", sanitizedData);

    // Reset form on success
    setFormData({
      name: "",
      email: "",
      subject: "",
      category: "",
      message: "",
    });

    setErrors({
      name: "",
      email: "",
      subject: "",
      category: "",
      message: "",
    });

    setSubmitStatus("success");
  } catch (error) {
    console.error("Form submission error:", error);
    setSubmitStatus("error");
  } finally {
    setIsSubmitting(false);
  }
};
```

### Explanation of the Code

The `handleSubmit` function is called when the user submits the contact form. It prevents the default form submission behavior, sets the submitting state, and performs comprehensive validation of the form data. If validation fails, it updates the errors state with appropriate messages. If validation passes, it simulates an API call to submit the form data and resets the form on success. It also handles any errors that occur during submission.
