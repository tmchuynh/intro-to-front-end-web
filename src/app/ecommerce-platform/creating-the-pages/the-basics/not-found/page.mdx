# 404 Not Found

## Table of Contents

## Purpose

The Not Found page is a standard page that informs users when they attempt to access a page that does not exist on the site. It should provide a friendly message and options to navigate back to the home page or other relevant sections of the site.

There are several ways to implement a Not Found page, but the most common approach is to create a dedicated page that is displayed when a user tries to access a non-existent route. This page can be styled to match the overall design of the site and can include links to help users find their way back.

For Next.js applications, you can create a custom 404 page by creating a file named `not-found.tsx` in the `app` directory. This file will automatically be used by Next.js to render the Not Found page when a user tries to access a route that does not exist.

## Content

Many websites have specific content that is displayed on their Not Found page. This content can include:

- A friendly message indicating that the page could not be found.
- A search bar to help users find what they are looking for.
- Links to popular pages or categories on the site.
- A link to the home page.
- A contact link for users to report the issue or get help.
- Social media links to encourage users to follow the site on various platforms.
- A link to the site's sitemap to help users navigate the site.
- A link to the site's privacy policy and terms of service.

### Example 1: Basic Not Found Page

```tsx
export default function NotFound() {
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50">
      <div className="text-center">
        <h1 className="text-9xl font-bold text-gray-200">404</h1>
        <h2 className="text-3xl font-semibold text-gray-800 mt-4">
          Page Not Found
        </h2>
        <p className="text-gray-600 mt-4 max-w-md mx-auto">
          Sorry, we couldn't find the page you're looking for. The page might
          have been removed or the URL might be incorrect.
        </p>
        <div className="mt-8">
          <Link
            href="/"
            className="bg-blue-600 text-white px-6 py-3 rounded-lg hover:bg-blue-700 transition-colors"
          >
            Go Back Home
          </Link>
        </div>
      </div>
    </div>
  );
}
```

#### Benefits

- Simple and Clean: Minimalist design that's easy to understand
- Fast Loading: No complex components or external dependencies
- Responsive: Works well on all device sizes
- Clear Call-to-Action: Single prominent button to return home

#### Disadvantages

- Limited Functionality: Only provides basic navigation back to home
- No Engagement: Doesn't offer alternative ways to find content
- Missed Opportunity: Doesn't leverage the moment to showcase site features

### Example 3: Not Found Page with Search Bar

```tsx
"use client";

import { useState } from "react";
import { FaSearch, FaMagnifyingGlass } from "react-icons/fa6";

export default function NotFoundWithSearch() {
  const [searchQuery, setSearchQuery] = useState("");

  const handleSearch = (e: React.FormEvent) => {
    e.preventDefault();
    if (searchQuery.trim()) {
      window.location.href = `/search?q=${encodeURIComponent(searchQuery)}`;
    }
  };

  return (
    <div className="min-h-screen bg-gray-900 text-white flex items-center justify-center">
      <div className="container mx-auto px-4 text-center">
        <div className="mb-8">
          <div className="inline-block p-4 rounded-full bg-red-500/20 mb-6">
            <FaMagnifyingGlass className="w-16 h-16 text-red-400" />
          </div>
          <h1 className="text-6xl font-bold mb-4">
            Page <span className="text-red-400">Not Found</span>
          </h1>
          <p className="text-xl text-gray-300 max-w-2xl mx-auto mb-8">
            We couldn't find what you were looking for. Try searching for
            something else, or explore our site using the navigation above.
          </p>
        </div>

        <form onSubmit={handleSearch} className="max-w-2xl mx-auto mb-12">
          <div className="relative">
            <input
              type="text"
              value={searchQuery}
              onChange={(e) => setSearchQuery(e.target.value)}
              placeholder="Search for products, articles, or pages..."
              className="w-full px-6 py-4 pl-14 text-lg bg-gray-800 border border-gray-700 rounded-full focus:outline-none focus:border-blue-500 focus:ring-2 focus:ring-blue-500/20 transition-all"
            />
            <FaSearch className="absolute left-5 top-1/2 transform -translate-y-1/2 text-gray-400 w-5 h-5" />
            <button
              type="submit"
              className="absolute right-2 top-1/2 transform -translate-y-1/2 bg-blue-600 hover:bg-blue-700 px-6 py-2 rounded-full transition-colors"
            >
              Search
            </button>
          </div>
        </form>

        <div className="flex flex-col sm:flex-row gap-4 justify-center">
          <Link
            href="/"
            className="bg-blue-600 hover:bg-blue-700 px-8 py-3 rounded-full transition-colors"
          >
            Go to Homepage
          </Link>
          <Link
            href="/contact"
            className="border border-gray-600 hover:border-gray-400 px-8 py-3 rounded-full transition-colors"
          >
            Contact Support
          </Link>
        </div>
      </div>
    </div>
  );
}
```

#### Benefits

- Immediate Solution: Users can search right from the error page
- Dark Theme: Modern, developer-friendly aesthetic
- Interactive: Engaging search functionality keeps users active
- Multiple Options: Provides both search and navigation alternatives

#### Disadvantages

- Search Dependency: Requires a working search system to be effective
- Client-Side: Needs JavaScript enabled to function properly
- Dark Design: May not fit all brand aesthetics or accessibility needs
