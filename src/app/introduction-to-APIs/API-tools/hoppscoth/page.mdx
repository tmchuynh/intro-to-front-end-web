## Introduction

[Hoppscotch](https://docs.hoppscotch.io/documentation/getting-started/introduction) is a lightweight, open-source API development ecosystem designed with ease of use and accessibility in mind. Built from the ground up, it provides all the functionality needed for API development with a minimalist, unobtrusive UI that works seamlessly across platforms.

Hoppscotch stands out as a **web-first** API development tool that requires no installation and works directly in your browser, while also offering desktop applications and CLI tools for comprehensive API testing workflows.

![Hoppscotch](https://mintlify.s3.us-west-1.amazonaws.com/hoppscotch/images/hoppscotch-light.png)

### Core Philosophy

Hoppscotch is built on several key principles:

- **Open Source & Auditable**: Source code is open and auditable, built with privacy and security in mind
- **Cross-Platform Accessibility**: Works on Web, Mac, Windows, Linux, and CLI with no installation required
- **Performance-Driven**: Built with performance in mind, designed to be seamless and instant
- **Community-Driven**: Built on top of open source technologies by the community, for the community
- **Developer-Focused**: Designed for teams to collaborate on APIs with ease
- **Keyboard-First Design**: Intuitive interface with comprehensive keyboard shortcuts
- **Security-Conscious**: Built with security in mind and designed to be safe and secure

## Using Hoppscotch

### Creating and Sending Requests

##### Basic Request Creation

1. **Open Hoppscotch** in your browser or desktop app
2. **Select the HTTP method** from the dropdown (GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS)
3. **Enter the API endpoint URL** in the request URL field
4. **Configure request details** including headers, parameters, authentication, and body
5. **Click the** <Button variant="tertiary" className="mx-1">Send</Button> **button** to execute the request
6. **Analyze the response** in the response panel showing status, headers, and body

##### Request Examples

###### Simple Get Request

```http
GET https://jsonplaceholder.typicode.com/posts/1
Content-Type: application/json
Accept: application/json
```

###### Post Request with JSON Body

```http
POST https://jsonplaceholder.typicode.com/posts
Content-Type: application/json

{
  "title": "My New Post",
  "body": "This is the content of my new post",
  "userId": 1
}
```

###### Authentication Examples

```http

## Basic Auth
```

**OAuth 2.0 Flow**:

```javascript
// Pre-request script for OAuth token refresh
const tokenUrl = pm.environment.get("tokenUrl");
const clientId = pm.environment.get("clientId");
const clientSecret = pm.environment.get("clientSecret");

if (!pm.environment.get("accessToken")) {
  // Implement OAuth flow
  pm.sendRequest(
    {
      url: tokenUrl,
      method: "POST",
      header: {
        "Content-Type": "application/x-www-form-urlencoded",
      },
      body: {
        mode: "urlencoded",
        urlencoded: [
          { key: "grant_type", value: "client_credentials" },
          { key: "client_id", value: clientId },
          { key: "client_secret", value: clientSecret },
        ],
      },
    },
    function (err, response) {
      if (response.code === 200) {
        const jsonData = response.json();
        pm.environment.set("accessToken", jsonData.access_token);
      }
    }
  );
}
```

##### Pre-request Scripts and Post-response Tests

Hoppscotch supports JavaScript scripting for dynamic request preparation and response validation.

**Pre-request Script Examples**:

```javascript
// Set dynamic timestamp
pm.environment.set("timestamp", new Date().toISOString());

// Generate random data
pm.environment.set(
  "randomEmail",
  `user${Math.random().toString(36).substr(2, 9)}@example.com`
);

// Calculate signature for API authentication
const message = pm.request.url.toString() + pm.environment.get("timestamp");
const signature = CryptoJS.HmacSHA256(
  message,
  pm.environment.get("secretKey")
).toString();
pm.environment.set("signature", signature);

// Conditional logic based on environment
if (pm.environment.get("env") === "production") {
  pm.request.headers.add({
    key: "X-Environment",
    value: "production",
  });
}
```

**Post-response Test Examples**:

```javascript
// Basic response validation
pm.test("Status code is 200", function () {
  pm.response.to.have.status(200);
});

pm.test("Response time is less than 500ms", function () {
  pm.expect(pm.response.responseTime).to.be.below(500);
});

// JSON schema validation
pm.test("Response has correct structure", function () {
  const jsonData = pm.response.json();
  pm.expect(jsonData).to.have.property("id");
  pm.expect(jsonData).to.have.property("name");
  pm.expect(jsonData.email).to.match(/^[^\s@]+@[^\s@]+\.[^\s@]+$/);
});

// Extract data for next request
pm.test("Extract user ID", function () {
  const jsonData = pm.response.json();
  pm.environment.set("userId", jsonData.id);
});

// Conditional testing
pm.test("Admin user has additional permissions", function () {
  const jsonData = pm.response.json();
  if (jsonData.role === "admin") {
    pm.expect(jsonData.permissions).to.include("admin_access");
  }
});
```

**Test Execution Summary**:

```
 Test Execution Results
┌─────────────────────────────────────────┐
│ Test Scripts: 8                         │
│ Test Suites: 12                         │
│ Test Cases: 45                          │
│ Requests: 25                            │
│ Test Duration: 00:00:15.234             │

   ───────────────────────────────────────┘

```

**Component Definitions**:

- **Test Scripts**: Number of `testScript` fields across all requests
- **Test Suites**: Number of `pw.test()` invocations in test scripts
- **Test Cases**: Number of `pw.expect()` assertions in test scripts
- **Requests**: Total HTTP requests executed
- **Test Duration**: Time spent executing test scripts
- **Requests Duration**: Time spent on HTTP requests
- **Pre-request Scripts**: Scripts executed before requests
