## How Data Types Work in TypeScript

Data types in TypeScript are a way to define the kind of data that can be stored in a variable. They help ensure that the data being used in your application is of the expected type, which can prevent errors and improve code quality.

### How They Differ from JavaScript

In JavaScript, variables can hold any type of data without strict type checking. TypeScript, on the other hand, enforces type annotations, allowing developers to specify the types of variables, function parameters, and return values. This leads to better tooling support and early error detection.

```typescript
let name: string = "John"; // String type
let age: number = 30; // Number type
let isActive: boolean = true; // Boolean type
let user: { name: string; age: number } = { name: "Alice", age: 25 }; // Object type
let numbers: number[] = [1, 2, 3]; // Array of numbers
let mixedArray: (string | number)[] = ["Alice", 30]; // Array with mixed types
```

### Lovely Errors with TypeScript's Type System

TypeScript's type system is designed to catch errors early in the development process. When you define a variable with a specific type, TypeScript will enforce that type throughout your code. If you try to use a value of a different type, TypeScript will raise an error.

```typescript
let count: number = 10;
count = "twenty"; // Error: Type 'string' is not assignable to type 'number'
```

```typescript
function display(value: string | number | boolean): void {
  console.log(value);
}
display("Hello"); // Valid
display(42); // Valid
display(true); // Valid
display({}); // Error: Argument of type '{}' is not assignable to parameter of type 'string | number | boolean'
```

```typescript
function add(a: number, b: number): number {
  return a + b;
}
add(5, 10); // Valid
add("5", "10"); // Error: Argument of type 'string' is not assignable to parameter of type 'number'
```

Even though TypeScript is a superset of JavaScript, it introduces a powerful type system that helps developers catch errors at compile time rather than runtime. This leads to more robust and maintainable code.

In the long run, using TypeScript can significantly reduce the number of bugs and improve the overall quality of your codebase.

## Why Use Data Types?

Using data types in TypeScript provides several benefits:

- **Type Safety**: Ensures that variables are used consistently and correctly.
- **Code Clarity**: Makes the code more readable and understandable.
- **Error Prevention**: Catches potential errors at compile time rather than runtime.
- **Intellisense Support**: Provides better autocompletion and documentation in IDEs.
