## Base Content

When you create a new Next.js project, it comes with a default `globals.css` file. This file is located in the `src/app/globals.css` directory. You can use this file to define global styles for your application.

### Adding Custom Styles

You can also add your own custom styles to the `globals.css` file. This is where you can define styles that are not covered by the default styles provided by Next.js or the theme variables.

```css
/* Custom styles */
:root {
  --warning: oklch(0.84 0.16 84);
  --warning-foreground: oklch(0.28 0.07 46);
}

.dark {
  --warning: oklch(0.41 0.11 46);

}

@theme inline {
  --color-warning: var(--warning);
  --color-warning-foreground: var(--warning-foreground);
}
```

## Custom Fonts

### Importing Fonts

You can also import fonts in the `globals.css` file. This is where you can define the fonts that will be used throughout the application.

```css
/* Importing fonts */
@import url("https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap");
@import url("https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap");
@import url("https://fonts.googleapis.com/css2?family=Open+Sans:wght@400;700&display=swap");
```

### Applying Fonts

You can apply the imported fonts to the body of the application. This is where you can define the default font for the application.

```css
/* Applying fonts */
body {
  font-family: "Inter", sans-serif;
  font-size: 16px;
  line-height: 1.5;
  color: var(--color-foreground);
  background-color: var(--color-background);
}
```

### Storing Fonts in `public/fonts`

You can also store your custom fonts in the `public/fonts` directory. This is where you can store your custom fonts and reference them in the `globals.css` file.

When you store your fonts in the `public/fonts` directory, you can reference them using the `url()` function in the `@font-face` rule. This allows you to use custom fonts in your application without relying on external font services.

You can define the `@font-face` rules in the `globals.css` file to specify the font family, source, weight, and style of the fonts you want to use.

Font files utilize several different formats to ensure compatibility across different browsers. The most commonly used formats are `woff2`, `woff`, and `ttf`. The `woff2` format is the most efficient and is recommended for modern browsers, while `woff` and `ttf` can be used as fallbacks for older browsers.

There are also variable fonts, which allow you to use a single font file that can adjust its weight, width, and other properties dynamically. This can reduce the number of font files you need to load, improving performance.

You can also specify the font weight and style to differentiate between regular and bold versions of the same font. This allows you to use different font weights in your application, such as normal and bold text.

```css
/* woff2 */
@font-face {
  font-family: "CustomFont";
  src: url("/fonts/custom-font.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
}

/* Variable Font */
@font-face {
  font-family: "CustomFontVariable";
  src: url("/fonts/custom-font-variable.woff2") format("woff2-variations");
  font-weight: 100 200 300 400 500 600 700 800 900;
  font-style: normal;
}

/* woff */
@font-face {
  font-family: "CustomFont";
  src: url("/fonts/custom-font.woff") format("woff");
  font-weight: 400;
  font-style: normal;
}

/* ttf */
@font-face {
  font-family: "CustomFont";
  src: url("/fonts/custom-font.ttf") format("truetype");
  font-weight: 400;
  font-style: normal;
}

/* otf */

  font-style: normal;
```

## Using `@layer` for CSS Organization

The `@layer` rule in CSS allows you to organize your styles into layers, which can help with specificity and maintainability. You can define different layers for different types of styles, such as base styles, component styles, and utility styles.

```css
/* Base styles */
@layer base {
  body {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
  }
}
/* Component styles */
@layer components {
  .button {
    background-color: var(--color-primary);
    color: var(--color-primary-foreground);
    padding: 0.5rem 1rem;
    border-radius: var(--radius);
    border: none;
    cursor: pointer;
  }
  .button:hover {
    background-color: var(--color-primary-foreground);
    color: var(--color-primary);
  }
}
```

This allows you to define styles in a way that is easy to read and maintain. You can also use the `@layer` rule to define custom layers for your application, such as `@layer utilities` for utility classes.

##### CAUTION

> The `@layer` rule is not supported in all browsers, so make sure to check the compatibility with your target browsers. It is primarily used in modern browsers and may not work in older versions.

## Using `@apply` for Tailwind CSS

If you are using Tailwind CSS, you can use the `@apply` directive to apply utility classes to your custom styles. This allows you to use Tailwind's utility-first approach while still defining your own custom styles.

```css
/* Using @apply for Tailwind CSS */
.button {
  @apply bg-primary text-primary-foreground px-4 py-2 rounded;
}
.button:hover {
  @apply bg-primary-foreground text-primary;
}
```

This allows you to leverage Tailwind's utility classes while still maintaining your own custom styles. You can use `@apply` to apply multiple utility classes to a single selector, making it easy to create consistent styles across your application.

##### CAUTION

> The `@apply` directive is not supported in all CSS preprocessors, so make sure to check the compatibility with your setup. It is primarily used in Tailwind CSS and may not work with other CSS frameworks or preprocessors.
